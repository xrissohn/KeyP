# 3분 데모 영상 스크립트

목표 길이: **2분 35초–2분 50초**. 공개 YouTube 영상, 음성 설명 필수. 화면을 1440p로 녹화하고 자막은 영어를 함께 넣는다.

## 0:00–0:18 — 문제와 한 문장 소개

화면: KeyP 홈 → 관심사 추가.

> 중요한 정보는 여러 언어와 플랫폼에 흩어져 있고, 검색 결과가 새 글이라고 해서 실제 사건도 새로운 것은 아닙니다. KeyP는 자연어 관심사 하나를 전 세계 공개 정보의 검증된 실시간 알림으로 바꿉니다. 슬로건은 “the signal, not the noise”입니다.

## 0:18–0:45 — 실제 입력

화면: 관심사에 아래 문장을 입력.

> “OpenAI의 새로운 에이전트 제품 발표와 실제 개발자 반응을 공식 발표 직후 알려줘. 이미 본 소식은 반복하지 마.”

화면: Planner UI에서 의도, 엔티티, 소스 계획이 만들어지는 장면.

> GPT‑5.6 IntentRefiner가 목표와 최신성 조건을 정제하고, QueryDecomposer가 검색을 공식·속보·소셜·영상·커뮤니티·한국의 여섯 전문 레인으로 나눕니다.

## 0:45–1:18 — 병렬 멀티 에이전트

화면: 개발자 도구 Network 또는 API 응답의 `steps`와 `metrics`를 펼친다. 여섯 Scout와 공개 어댑터, 네 Judge가 보이게 한다.

> 여섯 Scout는 Perplexity 웹 검색을 병렬로 실행하고, Bluesky·Hacker News·GDELT 및 선택형 SearXNG·RSS 어댑터도 동시에 동작합니다. X, Reddit, YouTube, TikTok, Instagram, Naver 같은 플랫폼은 공개 웹 결과만 검색하며, 비공개 계정에는 접근하지 않습니다.

> 이 구현은 OpenAI Agents SDK를 사용합니다. GPT‑5.6 에이전트 각각의 책임을 작게 제한해 한 모델이 검색과 판단을 동시에 하면서 생기는 오류를 줄였습니다.

## 1:18–1:48 — 신뢰성 데모

화면: 링크가 포함된 후보 → URL gate 단계 → 네 Judge 결과 → 최종 카드.

> 모든 URL은 실제 링크인지 확인하고 사설 IP, 위험한 리디렉션, 404와 soft‑404를 제거합니다. 그 다음 GPT‑5.6의 네 독립 Judge가 신뢰도, 관련성, 실제 사건의 최신성, 중복 여부를 각각 평가합니다. 최종 점수 계산과 오래된 사건 차단은 코드가 결정하므로 프롬프트가 제품의 안전 규칙을 우회할 수 없습니다.

## 1:48–2:12 — 사용자 가치

화면: 한국어로 편집된 알림 카드, 원문 링크 열기, 저장/좋아요, 푸시 알림 화면.

> MultilingualEditor가 사실을 추가하지 않고 한국어 또는 영어로 편집하며 원본 URL은 그대로 유지합니다. 사용자는 출처를 바로 열고, 저장하거나 평가할 수 있고, KeyP는 그 피드백으로 다음 알림을 개선합니다. 앱이 닫혀 있어도 푸시로 전달됩니다.

## 2:12–2:34 — 중복과 최신성 비교

화면: 같은 관심사를 다시 새로고침. `existingAlertSummaries`와 빈 결과 또는 새로운 사건만 남는 응답을 보여준다.

> 같은 이야기를 다른 매체가 다시 써도 의미 중복으로 제거합니다. 오늘 올라온 회고 기사라도 사건이 작년이면 최신 알림보다 뒤로 보내거나 차단합니다. 신뢰할 새 정보가 없다면 채우기용 결과를 만들지 않고 침묵합니다.

## 2:34–2:48 — Codex와 마무리

화면: GitHub Build Week 브랜치의 커밋, `src/services/swarm`, README, 테스트 통과 화면.

> Codex로 기존 저장소를 먼저 감사하고, Build Week 전후 경계를 분리한 브랜치에서 Agents SDK 통합, 공개 소스 어댑터, 테스트, 문서를 구현했습니다. KeyP는 더 많은 검색 결과가 아니라, 지금 나에게 중요한 다음 신호를 전달합니다.

## 녹화 전 체크

- `metrics.model`이 `gpt-5.6`인지 확인
- 여섯 Scout와 네 Judge가 응답 `steps`에 표시되는지 확인
- 결과 링크 2개 이상 직접 열기
- 두 번째 호출에서 첫 결과가 반복되지 않는지 확인
- 개인키, Replit Secrets, 요청 헤더, API 키가 화면에 나오지 않게 DevTools 정리
- 영상 길이 3분 미만, 음성 선명도, 공개 접근, 영어 자막 확인
